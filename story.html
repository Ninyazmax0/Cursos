La Linea del Tiempo

01
Capitulo I

Solo era una "Academia".

7 de Agosto, 2025

Todo empezó con un archivo index.html y una idea simple: crear una página para aprender programación. Sin nombre elegante, sin grandes planes. Solo "Academia". Diseño básico con luces azules, tema oscuro, y toda la información hardcodeada en el HTML como si fuera 1999.

"No sabía que esto era solo el primer borrador de algo mucho más grande. Un borrón con suerte."

02
Capitulo II

Era basura. Punto.

30 de Agosto, 2025

Después de tres semanas mirando esa página, llegué a una conclusión brutal: era una completa basura. El diseño era feo, la lógica era un desastre de condicionales imposibles, y cada vez que intentaba agregar algo nuevo, todo se rompía como un castillo de naipes bajo la lluvia.

Así que el 30 de agosto tomé la decisión más liberadora de mi vida: borrar todo y empezar de nuevo. Sin remordimientos, sin mirar atrás.

03
Capitulo III

Code Lingua nace.

4 de Septiembre, 2025

Cinco días después del genocidio de código, empecé de nuevo. Esta vez con herramientas modernas: Tailwind CSS para no morir con CSS vanilla, fondos de pantalla animados para darle vida al asunto, y una estructura más limpia que permitía respirar.

Le puse un nombre genérico: Code Lingua. Sonaba profesional, corporativo, importante. Aunque después descubrí que habían como 47 páginas exactamente con ese nombre. Ironía level: Máximo.

04
Capitulo IV

Amelia al rescate.

Mediados de Septiembre, 2025

Después de dos semanas trabajando solo, con ojeras que podían verse desde el espacio, me di cuenta de algo: necesitaba ayuda. Especialmente con el backend. Firebase era un misterio para mí.

Contacté a Amelia. Le mostré lo que tenía: un menú básico, 4 cursos funcionando a medias, y una desesperación visible. Le expliqué la idea. Y para mi sorpresa total, le gustó. Le gustó el caos.

Ella llegó con su expertise: 70% backend, 30% frontend. El complemento perfecto.

05
Capitulo V

Status Code 418.

Finales de Septiembre, 2025

"Code Lingua es demasiado genérico", dijo Amelia un día. Tenía razón. Existían docenas de páginas con ese nombre, todas igualmente olvidables.

Después de buscar nombres únicos durante horas, ella sugirió: Status Code 418. El famoso error HTTP "I'm a teapot". Un código de error que técnicamente no debería existir, pero existe porque el April Fools' de 1998 fue demasiado gracioso.

No queríamos ser otra página aburrida con 4 paletas de colores. Queríamos algo con personalidad.

06
Capitulo VI

Llevándolo a la nube.

Octubre, 2025

La academia funcionaba, pero todo el progreso se guardaba localmente. Si alguien cambiaba de dispositivo, perdía todo. Si el navegador se actualizaba, puff, todo se perdía. Eso no podía seguir así.

Amelia tomó el control del backend como un general toma una fortaleza. Configuró Firebase Firestore, reestructuró la arquitectura de datos, creó el sistema de autenticación, y convirtió todo en algo que sobrevivía entre sesiones.

07
Capitulo VII

Todo en vivo.

Noviembre, 2025

Amelia implementó sincronización en tiempo real. Abrió dos ventanas del navegador, completó un nivel en una, y casi al instante, la tabla de posiciones en la otra se actualizó sola. Sin refrescar. Sin recargar. Sin magia visible. Solo código bien hecho.

Agregamos un leaderboard en vivo, un sistema de logros que daba satisfacción instantánea, y una Sala de Trofeos para los que llegaban primero. Ya no era solo una plataforma de cursos. Era una experiencia.

08
Capitulo VIII

Cuando todo dejó de funcionar.

Finales de Noviembre, 2025

Después de migrar todo a la nube con fanfarrias y confeti imaginarios, algo salió terriblemente mal. El mapa de niveles no aparecía, los perfiles no cargaban, y había errores por todas partes como si un gato hubiera caminado sobre el teclado.

Pasamos horas revisando cada línea de código, tomando café como si fuera agua. Amelia encontró el primer problema: una función que nunca se estaba ejecutando. Lo arreglamos y los niveles aparecieron. Para la una de la mañana, todo funcionaba.

09
Capitulo IX

El rubí que faltaba.

Enero, 2026

Empezamos el año con una idea loca: no bastaba con Python y Web. Queríamos que Status Code 418 fuera una academia de verdad. Así que Amelia sugirió agregar Ruby.

El curso de Ruby creció hasta convertirse en algo impresionante. Con su sintaxis elegante y misteriosa, sus gemas brillantes, y esa filosofía de "hay más de una forma de hacerlo".

Y no paramos ahí. También agregamos Database, el curso de Introducción para absolute beginners, y mejoramos el de Web hasta que brilló como nunca antes.

10
Capitulo X

Vanidad 101.

Febrero, 2026

Los cursos estaban completos, el sistema funcionaba, pero algo faltaba. Personalidad. Expresión. Una forma de que cada usuario se sintiera único dentro de la academia.

Así que nacieron los cosméticos. Auras que brillan alrededor del avatar, fondos personalizados para el perfil, efectos visuales que hacen que cada usuario sea irrepetible.

"Los cosméticos no dan poderes", le dije a Amelia. "No", respondió ella, "dan estatus. Y el estatus es poder."

11

Abril, 2026
